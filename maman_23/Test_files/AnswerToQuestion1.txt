תשובות לשאלה 1:
א.לא נכון.
בשפת ANSI-C ניתן להגדיר פונקציות כסטטיות - static, ברגע שפונקציה מוגדרת כסטטית היא יכולה להקרא ע"י פונקציה אחרת רק מתוך הקובץ שבו היא הוגדרה מראש.
לדוגמא:
בקובץ fileA.c מוגדר:
#include <stdio.h>
static void funcA()
{
printf("inside funcA");
}

בקובץ fileB.cמוגדר:
#include <stdio.h>
static void funcA();
int main()
{
funcA();
return 0;
}

כאשר נקמפל את הקוד הבא ע"י makefile, נקבל שגיאת קומפילציה בטענה שהפונקציה funcA לא מוכרת ותודפס ההודעה - "'undefined reference to 'funcA"
הדבר מתרחש בעקבות כך שהפונקציה מוכרת רק בקובץ שבו הוגדרה מראש (fileA.c) ע"י כך שהיא סטטית.

ב. לפעמים נכון.
הפונקציה fsize מקבלת שמות קובץ\קבצים משורת הפקודה ומדפיסה את גודלם.
אם אחד הקבצים מוגדר כ"ספריה" Directory - הפונקציה תקרא לעצמה ברקורסיה - תפתח את הספריה ותדפיס בנוסף גם את גודל הקבצים שבתוך הספריה.
אם בשורת הפקודה לא הועברו שמות קבצים כמשתנים - הפונקציה תדפיס את גודל הספרייה הנוכחית.
